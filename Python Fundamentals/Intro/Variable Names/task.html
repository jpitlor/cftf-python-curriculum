Variable Names

Sometimes we want to set a variable but make sure it stays unmodified. For example, we may have a variable called gravity to represent the acceleration due to gravity that we want to stay the same throughout our program. To remind ourselves not to modify the value, we give constants a name in ALL CAPITALS, in this case GRAVITY. This will come up in a later chapter when we define colors.

Variable names must:

  • Start with a letter

Variable names MUST NOT:

  • Contain a space

Variable names SHOULD:

  • start with lower case (unless constant)
  • use underscores between words

Great variable names:

first_name
distance
ds9 # Although it's not clear what this is referring to

Allowed but not proper:

FirstName # starts with upper case; uses camelCase instead of underscore
firstName # Uses camelCase instead of underscore
X # starts with capital letter

Not allowed; won't compile:

first name # contains space
9ds # starts with number
%correct # starts with symbol
